# `.flow_flavor.json`

Source of truth for everything `flow flavor` does. Generated by `flow flavor
init` and edited by hand thereafter; every other flavor command reads (and
sometimes writes) this file.

The file lives at the project root.

## Full schema

```json
{
  "flavors": ["dev", "stage", "production"],
  "app_name": "Acme",
  "fields": {
    "baseUrl": "String",
    "debug": "bool",
    "maxRetries": "int"
  },
  "values": {
    "dev":        { "baseUrl": "https://dev.api.acme.com",     "debug": "true",  "maxRetries": "3" },
    "stage":      { "baseUrl": "https://stage.api.acme.com",   "debug": "true",  "maxRetries": "3" },
    "production": { "baseUrl": "https://api.acme.com",         "debug": "false", "maxRetries": "5" }
  },
  "app_config_path": "lib/core/config/app_config.dart",
  "use_separate_mains": true,
  "use_suffix": true,
  "android": {
    "application_id": "com.acme.app"
  },
  "ios": {
    "bundle_id": "com.acme.app"
  },
  "production_flavor": "production",
  "generate_scripts": false,
  "firebase": {
    "strategy": "unique_id_multi_project",
    "projects": {
      "dev":        "acme-app-dev",
      "stage":      "acme-app-stage",
      "production": "acme-app-production"
    }
  }
}
```

## Required fields

### `flavors`

Type: `List<String>` ¬∑ Required.

Every flavor your app supports. Names map directly to Android product flavors,
Xcode schemes, and `--dart-define=FLAVOR=<name>`.

- Must contain at least two entries.
- Names must match `[a-z][a-z0-9]*` ‚Äî lowercase, alphanumeric, no spaces.
- Conventional values: `dev`, `staging`, `production`.

### `app_name`

Type: `String` ¬∑ Required.

User-visible display name. Written into Android `AndroidManifest.xml`
`android:label` and iOS `Info.plist` `CFBundleDisplayName`.

- Auto-detected from `pubspec.yaml` `name:` during `init` (with title-casing).
- Override later by editing this field, then re-run `flow flavor init --from
  .flow_flavor.json` to re-sync platform manifests.

### `app_config_path`

Type: `String` ¬∑ Required.

Where the generated `AppConfig` Dart class is written. Path is relative to the
project root.

- Default: `lib/core/config/app_config.dart`.
- Must end in `.dart`. The parent directory is created if it doesn't exist.
- The file is overwritten on every `init` / `add` / `delete` / `replace`, so
  do not hand-edit it ‚Äî instead, regenerate.

### `use_separate_mains`

Type: `bool` ¬∑ Required.

- `true` ‚Äî one `lib/main_<flavor>.dart` per flavor. Each entry point sets
  `AppConfig.init(Flavor.<name>)` then calls `runApp(...)`. Recommended.
- `false` ‚Äî a single `lib/main.dart` reads `--dart-define=FLAVOR=...` and
  initialises the right flavor at runtime.

The choice affects what `flow flavor run` / `build` pass as `--target`.

### `use_suffix`

Type: `bool` ¬∑ Required.

Controls whether non-production flavors get a suffixed package ID.

- `true` ‚Äî non-production flavors append `.<flavor>` to the base ID:
  `com.acme.app.dev`, `com.acme.app.stage`, `com.acme.app`. **Required** if
  you want side-by-side installs on the same device.
- `false` ‚Äî all flavors share the base ID. Cannot install two flavors on the
  same device but stays consistent with App Store policy that you publish a
  single bundle.

The production flavor **always** uses the base ID, regardless of this setting.

### `android.application_id`

Type: `String` ¬∑ Required.

Base Android `applicationId`. Auto-detected from `android/app/build.gradle`
during `init`.

### `ios.bundle_id`

Type: `String` ¬∑ Required.

Base iOS `CFBundleIdentifier`. Auto-detected from `ios/Runner.xcodeproj`
during `init`.

### `production_flavor`

Type: `String` ¬∑ Required.

The flavor that should keep the base package ID (no suffix) and serve as the
canonical "release" identity. Must be a value present in `flavors`.

If you delete this flavor, `flow flavor delete` will prompt you to nominate a
replacement.

## Optional fields

### `fields`

Type: `Map<String, String>` ¬∑ Default `{}`.

The schema of your runtime `AppConfig` class. Keys are field names; values
declare the Dart type. Supported types:

| Declared type | Dart type | Notes |
|---|---|---|
| `String` | `String` | All values become strings unless quoted differently |
| `bool` | `bool` | Values must be `"true"` or `"false"` in `values` |
| `int` | `int` | Parsed as integer |
| `double` | `double` | Parsed as `double.parse` |

Generated Dart class for the schema above:

```dart
// GENERATED CODE - DO NOT MODIFY BY HAND
enum Flavor { dev, stage, production }

class AppConfig {
  static late Flavor flavor;
  static late String baseUrl;
  static late bool debug;
  static late int maxRetries;

  static void init(Flavor f) {
    flavor = f;
    switch (f) {
      case Flavor.dev:
        baseUrl = 'https://dev.api.acme.com';
        debug = true;
        maxRetries = 3;
        break;
      // ‚Ä¶
    }
  }
}
```

### `values`

Type: `Map<String, Map<String, dynamic>>` ¬∑ Default `{}`.

Per-flavor concrete values for each `fields` key. Outer keys must match
entries in `flavors`; inner keys must match entries in `fields`. Strings only
‚Äî `flow` parses them according to the declared type.

If a flavor is missing a value for a defined field, validation fails:

```terminal
‚ùå flow: invalid config at ".flow_flavor.json"
   ‚Üí values["dev"] is missing key "baseUrl"
```

### `generate_scripts`

Type: `bool` ¬∑ Default `false`.

When `true`, `flow flavor init` writes shell scripts under `scripts/` for each
common build target. Useful for CI pipelines that prefer to invoke shell
scripts rather than the Dart CLI.

### `firebase`

Type: `Object | null` ¬∑ Default `null` (Firebase setup skipped).

Configures `flow flavor firebase`. Two sub-fields:

#### `firebase.strategy`

Type: `String` ¬∑ Required if `firebase` is present.

One of:

| Value | Meaning |
|---|---|
| `shared_id_single_project` | All flavors share one Firebase project and one package ID. `use_suffix` must be `false`. |
| `unique_id_single_project` | All flavors live in one Firebase project but each registers its own platform app under its suffixed package ID. `use_suffix` must be `true`. |
| `unique_id_multi_project` | Each flavor is registered in its own Firebase project. Recommended for true environment isolation. |

#### `firebase.projects`

Type: `Map<String, String>` ¬∑ Required if `firebase` is present.

Maps each entry in `flavors` to a Firebase project ID (the one you set in the
Firebase console URL). For single-project strategies, all flavors should point
to the same project ID.

```json
"firebase": {
  "strategy": "unique_id_multi_project",
  "projects": {
    "dev":        "acme-app-dev",
    "stage":      "acme-app-stage",
    "production": "acme-app-production"
  }
}
```

## Validation

Every flavor command validates the file on load. Missing required fields,
mismatched flavor names between `flavors`/`values`, and unknown Firebase
strategies all fail fast with a clear pointer to the offending JSON path:

```terminal
‚ùå flow: invalid config at ".flow_flavor.json"
   ‚Üí "production_flavor" must be one of the declared flavors: [dev, stage, production]
```

:::warn Don't hand-edit generated artifacts
The `AppConfig` Dart file, `launch.json`, and Android / iOS configuration are
all regenerated by `flow flavor` commands. Change `.flow_flavor.json` and run
the appropriate command ‚Äî never edit generated files directly, your changes
will be overwritten.
:::
